# Horae - 時光記憶 v1.10.1 | SillyTavern 記憶增強插件

![Image](https://github.com/SenriYuki/SillyTavern-Horae/blob/main/HoraeLogo.jpg)

> *Horae（荷賴）— 希臘神話中掌管時序的女神*

長篇 RP 玩家的老毛病你一定遇到過 —— AI 的記憶約等於金魚：昨天的事說成今天早上，連幾天前發生的事都永遠說是昨天；上一幕穿校服下一幕突然便服；NPC關係倒置；送出去的禮物憑空消失，丟掉的東西又回到手裡。

**Horae 用結構化的時間錨點，給你的 AI 裝上一本可靠的記憶賬本。**

---

## 亮點功能

**RPG 模式 —— 血條、技能、聲望、裝備、等級、貨幣一目瞭然**
為西幻 / 修真 / 戰鬥向角色卡量身打造的全模組化 RPG 系統。各子系統（屬性條、多維屬性、技能、聲望、裝備、等級、貨幣）均為**獨立可選開關**，按需組合。角色卡內資訊採用**標籤分頁**展示（屬性/技能/裝備/聲望/等級/貨幣），僅顯示已開啟的模組，節約空間。**聲望系統**支援客製化勢力分類和數值範圍。**裝備系統**支援客製化格位與屬性值追蹤。**等級系統**追蹤角色等級和經驗值，經驗條進度可視化。**貨幣系統**支援客製化幣種和兌換比率（如 1000銅=10銀=1金），AI 可透過增量或絕對值修改。所有配置均可導出/導入 JSON。關閉時不注入提示詞、不消耗 Token。

**向量記憶引擎 —— 找回被摺疊的細節（新）**
專為搭配「自動摘要&隱藏」設計的智慧外腦！幾十萬字長篇壓縮後細節全丟？現在當對話觸及歷史事件時，插件會自動從被隱藏的舊時間線中精準召回相關片段。全程依託 Web Worker 純本地運算，零 API 消耗，滑順不卡頓界面（首次執行需下載 30-60MB 模型快取，提供中文最佳化的 bge 和多語言 e5 雙模型可選⚠ 2GB 以下手機或雲酒館建議別用）。

**新手導航 —— 第一次用也不迷路（新）**
首次使用 Horae 的用戶會自動觸發互動式導覽，逐步介紹各功能區域和設定項。從基本概念到進階功能（上下文深度、注入位置、客製化提示詞等）一次講清。老用戶可在設定中隨時「重新開始教學」。

**自助美化工具 —— 不會寫 CSS 也能客製化（新）**
為不會寫代碼的用戶設計的可視化美化面板。用色相條、飽和度/亮度滑桿快速調色，日夜模式一鍵切換，還能導入圖片 URL 裝飾抽屜頭部、背景和底部訊息欄。所有修改即時預覽，滿意後保存即可導出分享。當然，也保留了主題 JSON 導入/導出和手寫 CSS 給進階用戶。（⚠️做 Horae 插件的美化不用跟我要授權，請隨意。）

**場景記憶 —— 酒館不再一夜變樣（新）**
AI 描寫同一個地點，上一回有壁爐這一回沒了？場景記憶會記錄地點的固定物理特徵，下次進入同一場景時自動注入。智慧配對地名（「酒館·大廳」自動回退到「酒館」），只發送配對到的那一條描述，未命中時零注入。

**情緒 & 關係網路 —— AI 讀懂人際場（新）**
情緒追蹤讓 AI 記住角色目前的心理狀態，不再前一秒哭後一秒笑。關係網路記錄人物之間的關係類型和變化，再也不會把敵人寫成朋友。兩者都是變化驅動，沒變化時零輸出。

**自動摘要 —— 長篇自動省 Token**
聊了幾百樓還在全量發送？自動摘要在後臺自動壓縮舊訊息為一段摘要，原始樓層 `/hide` 省 Token。摘要和原始時間線可隨時一鍵切換，所見即所發。還能配一個低成本的**獨立 API** 做真並行，摘要生成完全不佔主連接。

**向量記憶 —— 讓 AI 回想起被隱藏的細節（新）**
搭配自動摘要使用的智慧回憶系統。摘要壓縮後的舊訊息細節不會遺失——當對話涉及歷史事件時，插件會自動從被隱藏的時間線中精準召回相關片段注入上下文。全程**本地運算**，不消耗額外 API。支援中文最佳化和多語言兩種模型選擇。

**AI 智慧摘要**
舊檔案沒用過 Horae？一鍵批量掃描全部歷史訊息，AI 自動提取劇情事件和物品變化，生成完整的時間線。支援客製化每批 Token 上限符合不同模型，不滿意可一鍵撤銷。

**告別"永遠的昨天"**
插件自動計算相對時間，注入"昨天"、"上週三"、"上個月15號"這樣的生活化表述。AI 終於能分清前天和上週了。支援現代、歷史、奇幻架空等多種日曆體系。

**衣服不再亂穿**
每位角色的目前穿著被鎖定記錄，且只發送在場角色的服裝。角色不會再莫名其妙換昨天的衣服，也不會把不在場的人的衣服發給 AI 浪費 token。

**NPC 不會越寫越糊**
外貌、個性、關係等資料欄獨立追蹤。年齡還會隨劇情時間自動推進。嚴格訂立了關係的 prompt，不再上一秒 Char 欠 User 錢，下一秒 AI 輸出說是 User 欠 Char 錢來顛倒關係。

**物品欄終於可靠了**
帶唯一編號的物品系統，分一般/重要/關鍵三級。智慧解析數量變化（5斤→4斤），自動檢測"已消耗"等狀態並移除。

**待辦事項防遺忘**
AI 可以自動記錄劇情中的約定和伏筆，完成後自動刪除。再也不用擔心 AI 忘記兩週前約好的約會。

**寶貝輔食級客製化表格**
Excel 風格的自由表格，想加行加行想加列加列，配上填寫提示詞讓 AI 自動填。課程表、角色關係、任務清單……你能想到的都能做。刪除訊息時 AI 填的內容自動回退，你自己填的始終保留。

**變化驅動，省 Token**
AI 只輸出本回合變化的資訊，不再每次重複整個狀態。每種數據都有明確的觸發規則，減少無效輸出。**不用額外耗費生成次數** —— 插件在 AI 正常回覆時自動解析記憶節點，零額外開銷。

**截斷/改寫也不怕**
中途被截斷或自己手動改寫了部分劇情？底部欄內置 **AI 分析**功能，只需一次生成就能智慧補全該條訊息的記憶節點。

**界面簡單，新手友好**
底部欄一目瞭然，頂部面板分頁清晰。無需複雜配置，安裝即用。

**發送內容自由控制**
在設定中自由選擇哪些數據發送給 AI、哪些不發送 —— 不需要角色追蹤？關掉。不需要物品欄？關掉。按需組合，節省 token。

## v1.3.0 更新內容

- **正則置底**：插件正則啟動時自動移至列表末尾，減少與其他正則衝突
- **時間線多選**：時間線事件支援長按多選批量刪除（避免誤觸做了較長的觸發時間）

## v1.5.0 更新內容

本次更新圍繞 RP「精細度」與「長篇省 Token」展開。

### 新增功能

- **情緒/心理狀態追蹤**：追蹤在場角色的情緒變化（如「緊張/不安」「開心/期待」）。底部欄自動顯示目前角色的情緒標籤。**Token 消耗極低**——AI 僅在情緒發生明顯變化時才輸出 `mood:` 標籤（通常 10-20 token），無變化時零輸出。設定中可開關
- **關係網路**：記錄角色之間的關係（朋友、戀人、上下級、宿敵等）。NPC 頁面底部顯示關係列表，支援手動編輯。底部欄的關係展示**完全由插件從資料庫讀取彩現，不消耗 AI 輸出 Token**。AI 端採用變化驅動：僅在關係變化時輸出 `rel:` 標籤，已有關係無變化時零輸出。設定中可開關
- **場景記憶**：記錄地點的固定物理特徵，保持跨回合場景描寫一致。**智慧檢索**——插件根據目前地點名配對已記錄的場景描述，僅發送配對到的那一條（通常 50-150 字），而非把所有地點記憶全部發送。未配對到任何地點時零注入。支援複合地名回退配對（如「酒館·大廳」→「酒館」）
- **自動摘要 & 隱藏**：設定中開啟後，當未壓縮訊息數超過閾值時自動觸發全文摘要。摘要完成後原始訊息樓層被 `/hide` 省 Token，時間線中出現摘要卡片。可隨時點擊按鈕在「摘要視圖」和「原始時間線」之間切換，所見即所發
- **獨立 API（真並行）**：自動摘要可配置獨立的 OpenAI 相容端點（填入 API 地址 / 密鑰 / 模型名稱即可）。摘要生成透過直接 HTTP 請求，完全不佔用酒館主連接，實現與 AI 回覆的真並行。支援 OpenAI、DeepSeek、OpenRouter 等所有相容端點
- **劇情壓縮取消按鈕**：事件壓縮和全文摘要均新增取消按鈕，點擊後立即中斷後端 API 請求（與 AI 智慧摘要的取消按鈕一致）
- **表格分支繼承**：開分支或回退訊息時，表格數據只繼承到該節點為止的 AI 填寫和用戶手動編輯。例如在 #10 開分支，只繼承 #10 之前的表格數據，不會帶入後續填寫

### 改進

- **全域表格數據按卡分離**：全域表格的結構（表頭、名稱、提示詞、鎖定）跨角色卡共享，但填寫數據按角色卡獨立存儲。切換卡片不再繼承其他卡的表格數據
- **智慧記憶壓縮**：在時間線多選模式下可將多條事件交給 AI 壓縮合併為一條摘要，支援摘要視圖與原始時間線雙向切換
- 系統提示詞編輯器中的預設文字會根據目前啟用的功能動態更新（如啟用關係網路後 `<horae>` 標籤格式區自動出現 `rel:` 行）

### UI / UX

- 修復 `--horae-surface` 未定義變數，改用 `--horae-bg-secondary`。**已有的客製化美化不受影響**
- THEME.md 補充了新增選擇器說明文件

### ⚠️ 更新提醒

如果你之前**客製化過系統注入提示詞**，更新到 v1.5.0 後請進入設定 → 客製化提示詞，點擊各欄位旁的 **🔄 重置** 按鈕恢復為最新預設值。因為新增的場景記憶、情緒追蹤、關係網路功能需要在提示詞中聲明對應的標籤格式（`scene_desc:`、`mood:`、`rel:`），舊的客製化提示詞中缺少這些聲明會導致 AI 不輸出新標籤。

如果你沒有客製化過提示詞（使用預設值），則無需操作，插件會自動使用最新的預設提示詞。

## v1.7.0 更新內容

本次更新新增向量記憶系統，讓自動摘要的用戶不再因壓縮而遺失細節。

### 新增功能

- **向量記憶 (Beta)**：基於本地 AI 模型的智慧回憶系統，專為搭配「自動摘要 & 隱藏」設計。當對話中涉及歷史事件時，插件會自動從被隱藏的舊訊息中精準召回相關時間線片段，注入上下文供 AI 參考
  - **全程本地運算**：使用瀏覽器內置 Web Worker 執行輕量 AI 模型，不消耗額外 API 額度。首次使用需下載約 30-60MB 的模型資料（之後會快取在瀏覽器中）
  - **雙模型可選**：`bge-small-zh-v1.5`（中文最佳化）和 `multilingual-e5-small`（多語言支援），可按需選擇
  - **智慧檢索**：結合意圖識別、結構化查詢和語義搜索三層檢索策略。例如「提及和角色初次見面」會直接搜尋該角色首次出現的訊息，無需依賴模糊配對
  - **上下文關聯**：命中訊息的前後相鄰事件也會被自動帶入，還原完整的事件脈絡
  - **全文回顧**：高置信度的召回結果可發送原始正文（自動過濾思維鏈），讓 AI 獲得完整的敘事細節而非僅時間線摘要。條數和閾值均可客製化，設為 0 可關閉
  - **摘要隔離**：自動排除摘要內容，只使用原始時間線數據參與檢索，避免摘要重複注入

## v1.9.0 更新內容

### 自動摘要 & 副 API

- **副 API 多輪結構改造**：重構摘要請求的訊息結構，採用 system/assistant/user 多輪交替格式，內嵌 assistant 確認回覆和預填充（prefill），大幅提升 NSFW 內容的摘要透過率。同時自動注入酒館預設中的主提示詞、NSFW 允許提示詞和破限提示詞
- **模型下拉選單**：副 API、Embedding、Rerank 三處模型設定均從文字輸入改為下拉選單。填好 API 地址和密鑰後，點擊重新整理按鈕即可拉取可用模型列表，無需手動輸入模型名
- **自動摘要警告最佳化**：不再因部分樓層缺少時間戳就完全阻止摘要。改為詳細顯示哪些樓層缺少時間戳或時間線事件（如 `缺時間戳: #5, #8, #12`），僅在超過 50% 缺失時彈出警告提醒，但不阻止摘要執行

### 向量記憶

- **全文剔除標籤**：向量設定中新增「全文剔除標籤」輸入框。用戶可填入客製化標籤名（如 `snow, theater`），全文回顧和 Rerank 全文精排時會將這些標籤包裹的內容整段移除，避免小劇場/旁白等非劇情內容被 AI 召回
- **番外訊息隔離**：標記為「番外」（`_skipHorae`）的訊息不再被向量索引和召回，確保小劇場模式的內容不會汙染正式劇情的記憶檢索

## v1.10.0 更新內容

本次大版本更新全面重構 RPG 系統，新增 6 個 RPG 子模組、據點經營系統、裝備多種族模範等功能。

### RPG 模式

- **子模組獨立開關**：裝備欄、聲望、等級、貨幣、據點各為獨立 checkbox，按需組合。關閉時零 Token 消耗
- **角色卡標籤分頁**：角色詳情區改為分頁標籤（屬性/技能/裝備/聲望/貨幣），僅顯示已開啟的模組
- **只顯示在場角色**：頂部角色卡和底部 HUD 根據 `characters_present` 過濾，離場角色數據保留但不顯示

### 新增功能

- **等級/經驗值系統**：AI 透過 `level:` / `xp:` 標籤輸出等級變化。經驗值公式：更新所需 = 等級 × 100，等級差 ≥10 的低級挑戰僅得 1 點經驗。手動編輯等級時自動計算所需經驗，無需手動輸入
- **貨幣系統**：客製化幣種和兌換比率。每個幣種可設定 emoji 圖示（如 🪙💎），在 HUD 和角色卡中顯示。AI 透過 `currency:歸屬|幣名=±增量` 修改，僅接受已註冊幣種。配置可導出/導入
- **聲望系統**：客製化勢力聲望分類和數值範圍，支援細項（子維度）。刪除防復活，導出/導入 JSON
- **裝備欄系統**：按角色獨立的格位配置與屬性值追蹤
  - **按角色獨立格位**：每個角色擁有獨立的裝備格位配置（莉莉絲用人類模範、雷格用獸人模範），互不干擾
  - **多種族模範**：內置人類、獸人（含尾部）、有翼族（含翅膀）、馬人（馬甲/馬蹄鐵）、拉彌亞（無腿部/足部）、惡魔（含角飾/翅膀/尾部）六種預設模範
  - **自存模範**：可將角色目前格位配置保存為新模範，隨時加載/刪除
  - **靈活配置**：加載模範後仍可為單個角色增刪格位，不影響其他角色
  - **穿卸追蹤**：AI 透過 `equip:` / `unequip:` 標籤記錄變化，插件按角色格位檢查有效性和數量上限
  - **手動管理**：支援手動添加裝備（選擇格位、輸入名稱和屬性）和卸下裝備
  - **導出/導入**：裝備格位配置可導出 JSON 分享
- **據點/基地系統**（新）：類似場景記憶的多層級樹形結構，用於管理角色的據點、領地、設施
  - **樹形層級**：支援無限巢狀子節點（如 莊園 > 鍛造區 > 鍛造爐）
  - **等級追蹤**：每個節點可設定等級
  - **AI 聯動**：AI 透過 `base:路徑=等級` 或 `base:路徑|desc=描述` 自動更新據點資訊，路徑用 `>` 分隔層級
  - **手動管理**：可手動添加/編輯/刪除節點，支援導出/導入
